# Policy Research Plan Example

免责声明：本文档仅用于 AIRS 工程开发与研究质量控制，不构成投资建议，不提供荐股、自动交易、交易指令、目标价或收益承诺。

## Goal Analysis

目标是分析半导体政策变化对产业链的研究路径，Planner 将其识别为 Policy Research。

## Scope

范围包含政策文本、监管事件、产业链影响、公司披露和反方情景。范围排除政策效果确定性判断。

## Required Connectors/Methodologies/Skills/Runtime

Connectors：policy、regulatory、market_news。Methodologies：policy-driven、event-impact、counter-consensus。Skills：news、evidence、risk、report。Runtime 必须等待 Planner Gate。

## Expected Evidence/KG

Evidence Plan 需要政策原文、监管公告、媒体报道和公司公告。KG 连接政策、行业、供应链节点、证据和风险。

## Execution Order

goal_analysis -> scope -> methodology_selection -> connector_plan -> evidence_plan -> skill_plan -> knowledge_graph_plan -> score_plan -> report_plan -> review_plan -> runtime_plan。

## Estimated Cost/Time

预计 81 分钟，成本单位约 23，最大并发 3。

## Expected Deliverables

政策研究计划、事件链、Evidence Plan、KG Plan、Scorecard Plan、报告提纲。

## Confidence

政策文件可得时置信度较高，但政策执行路径和市场反应需要标注不确定性。

## Risks

政策解读偏差、执行细则未落地、事件影响被过度外推。
