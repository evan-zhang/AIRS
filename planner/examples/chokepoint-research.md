# Chokepoint Research Plan Example

免责声明：本文档仅用于 AIRS 工程开发与研究质量控制，不构成投资建议，不提供荐股、自动交易、交易指令、目标价或收益承诺。

## Goal Analysis

目标是分析 HBM 供给瓶颈和反方证据，Planner 将其识别为 Chokepoint Research。

## Scope

范围包含瓶颈节点、替代路径、扩产难度、反方证据和缺失数据。范围排除确定性供需判断。

## Required Connectors/Methodologies/Skills/Runtime

Connectors：industry_report、company_filings、expert_view。Methodologies：supply-chain-chokepoint、counter-consensus、risk。Skills：supply-chain、evidence、risk、verification。Runtime 入口必须是 Planner Plan。

## Expected Evidence/KG

Evidence Plan 需要支持瓶颈的证据、削弱瓶颈的反方证据和缺失证据。KG 需要呈现瓶颈节点、替代边和风险节点。

## Execution Order

goal_analysis -> scope -> methodology_selection -> connector_plan -> evidence_plan -> skill_plan -> knowledge_graph_plan -> score_plan -> report_plan -> review_plan -> runtime_plan。

## Estimated Cost/Time

预计 81 分钟，成本单位约 23，最大并发 3。

## Expected Deliverables

卡点研究计划、证据链规划、KG 规划、风险复核路径、Review Checklist。

## Confidence

计划置信度为中等偏高，但取决于产能、良率和价格证据。

## Risks

专家观点可验证性不足、扩产数据滞后、替代路径低估、反方证据强度不足。
