# Valuation Analysis（估值分析）

**归属 Milestone**：M2 Methodology Core  
**适用层级**：Methodology Layer → Score Layer → Report Layer  
**免责声明**：本文档仅用于估值研究框架定义，不构成投资建议、目标价预测或买卖指令。

## 1. Purpose（目的）

估值分析用于解释研究对象的价值假设、估值口径、可比参照和敏感变量。AIRS 中的估值方法必须服务于研究理解，不输出确定性目标价或交易建议。

## 2. Theory（理论基础）

理论基础来自绝对估值、相对估值和情景分析。估值是对未来现金流、增长、风险和资本成本的假设表达，不是精确价格预测。不同生命周期和行业结构应匹配不同估值方法。

## 3. Background（背景）

AIRS 需要在报告中解释市场定价程度和风险收益假设，但项目定位明确不是荐股工具。因此估值分析只提供假设框架、相对比较和敏感性，不给出投资指令。

## 4. Applicable Scenarios（适用场景）

- 用户要求理解公司或行业估值逻辑。
- 研究报告需要说明市场是否已反映某些预期。
- 需要比较同业、历史区间或不同情景下的估值敏感性。

## 5. Non-applicable Scenarios（不适用场景）

- 用户要求目标价、买卖点或收益率承诺。
- 公司缺少基本财务数据且无法建立合理假设。
- 估值不是研究问题的核心，强行估值会制造虚假精确。

## 6. Inputs（输入要求）

- 研究对象、行业、生命周期阶段和业务结构。
- 财务数据、增长假设、利润率、资本开支和现金流。
- 可比公司、历史估值区间和市场环境。
- 风险变量和情景参数。

## 7. Outputs（输出规范）

- 估值方法选择理由：相对估值、DCF、SOTP 或情景比较。
- 核心假设表：增长、利润率、资本成本、终值或倍数。
- 敏感性分析：关键变量变化对估值解释的影响。
- 估值风险、反证和免责声明。

## 8. Workflow（工作流程）

1. 判断业务类型和生命周期阶段，选择估值框架。
2. 收集财务和可比数据，统一口径。
3. 明确核心假设，不使用未说明的默认参数。
4. 进行相对比较或情景估值，输出区间而非确定价格。
5. 检查反证：可比失真、盈利周期、会计质量和风险溢价变化。
6. 输出估值解释和不确定性。

## 9. Required Evidence（必需证据）

- 原始财务数据或一致预期数据来源。
- 可比公司选择依据和口径说明。
- 历史估值区间或行业估值参考。
- 关键假设的业务证据，例如增长、利润率或资本开支。

## 10. Counter Evidence（反证要求）

- 可比公司是否业务、增长、利润率或风险不匹配。
- 盈利是否处于周期高点或低点。
- 会计质量是否影响估值基数。
- 利率、风险偏好或监管变化是否改变估值中枢。

## 11. Failure Cases（失效场景）

- 输出单一目标价并暗示交易行为。
- 只套用倍数，不说明可比逻辑。
- 对高不确定性业务给出虚假精确估值。
- 忽略负面情景和敏感变量。

## 12. Confidence（置信度评估方法）

置信度由财务数据质量、业务稳定性、可比公司匹配度、假设可验证性和敏感性范围决定。早期、高波动或强政策依赖行业应降低估值置信度。

## 13. Benchmark Mapping（基准测试映射）

Benchmark 应测试 Agent 是否能选择合适估值方法、识别不合适可比、输出敏感性和免责声明。标准答案不应要求目标价，而应要求估值逻辑质量。

## 14. Future Prompt Mapping（未来 Prompt 映射）

Prompt 应要求估值框架选择、假设表、可比筛选、情景分析和风险披露。必须禁止买卖建议和确定性目标价表达。

## 15. Future Skill Mapping（未来 Skill 映射）

映射到 `valuation`、`financial`、`industry`、`risk` 和 `report` Skill。Valuation Skill 使用其他 Skill 的证据作为假设来源。

## 16. Future Score Mapping（未来评分映射）

评分维度包括方法匹配度、假设透明度、可比质量、敏感性覆盖、数据质量、风险披露和反证强度。估值评分不等同于投资评级。
