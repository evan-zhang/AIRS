# Evidence Chain Analysis（证据链分析）

**归属 Milestone**：M2 Methodology Core  
**适用层级**：Methodology Layer → Evidence Layer → Evaluation Layer  
**免责声明**：本文档仅定义研究证据组织方式，不构成投资建议，不提供任何交易动作建议。

## 1. Purpose（目的）

证据链分析用于把分散证据组织成可追溯、可验证、可反驳的推理链。该方法论要求每个结论都能回溯到证据卡，并标注证据等级、推理关系、缺口和反方证据。

## 2. Theory（理论基础）

理论基础来自证据推理、因果链分析和审计追踪。研究结论不是单一证据的堆砌，而是由多个证据通过因果、相关、对照、排除和时间先后关系构成的链条。

## 3. Background（背景）

AIRS 的核心原则是证据驱动。后续 M3 Evidence Engine 需要统一证据卡、证据链和证据质量评价，因此 M2 先定义方法论，确保所有研究 Skill 均可按同一证据标准输出。

## 4. Applicable Scenarios（适用场景）

- 任意需要形成研究结论的 AIRS 工作流。
- 报告中存在关键判断、评分或风险结论时。
- 需要检查结论是否由充分证据支持，或需要定位证据缺口。

## 5. Non-applicable Scenarios（不适用场景）

- 用户只要求解释概念，不形成研究判断。
- 数据完全不可得且无法建立最小证据链。
- 用户要求根据内部消息、传闻或未公开信息形成结论。

## 6. Inputs（输入要求）

- 研究命题或待验证假设。
- 证据卡集合，包含来源、时间、类型、摘要和可信度。
- 推理目标：支持、反驳、比较或解释。
- 证据质量规则：A级至D级及其使用限制。

## 7. Outputs（输出规范）

- 主证据链：结论所依赖的最短可解释链条。
- 辅助证据链：用于增强或交叉验证的证据。
- 反证链：可能削弱结论的证据及其强度。
- 证据缺口和后续验证清单。

## 8. Workflow（工作流程）

1. 把研究结论拆成可验证命题。
2. 为每个命题绑定证据卡，并标注证据等级。
3. 判断证据之间的关系：因果、相关、佐证、冲突或背景。
4. 检查链条是否存在断点、循环论证或单一来源依赖。
5. 引入反证并评估其对结论的削弱程度。
6. 输出证据链图谱、置信度和缺口。

## 9. Required Evidence（必需证据）

- 每个核心结论至少需要一个A级或B级证据；若没有，必须解释原因并降低置信度。
- 关键事实需要来源、发布时间和可复核路径。
- 跨来源交叉验证优先于单一来源重复引用。
- 引用观点类证据时必须标注其观点属性。

## 10. Counter Evidence（反证要求）

- 搜索或构造与主结论相反的数据、案例或逻辑。
- 标注反证是否来自同等或更高质量来源。
- 说明反证削弱的是事实前提、推理关系还是结论外推。
- 若没有找到反证，也要说明检索范围和剩余不确定性。

## 11. Failure Cases（失效场景）

- 只有结论和引用，没有推理关系。
- 多个证据来自同一原始来源却被误当作多源验证。
- 选择性引用支持性证据，忽略明显反证。
- 用过期数据支持当前判断。

## 12. Confidence（置信度评估方法）

置信度由证据等级、来源独立性、时效性、链条完整性、反证强度和推理透明度决定。存在关键断点时，即使证据数量多，也不得给高置信度。

## 13. Benchmark Mapping（基准测试映射）

Benchmark 应测试 Agent 能否从混杂资料中抽取证据卡、建立链条、识别冲突证据和给出证据缺口。评分重点是可追溯性和反方证据处理质量。

## 14. Future Prompt Mapping（未来 Prompt 映射）

Prompt 应要求输出结构化证据卡、证据关系表、反证表和置信度理由。不得允许“综合判断”“市场认为”等无来源表达支撑核心结论。

## 15. Future Skill Mapping（未来 Skill 映射）

直接映射到 `evidence` Skill，并为 `supply-chain`、`financial`、`valuation`、`risk` 和 `report` Skill 提供统一证据底座。

## 16. Future Score Mapping（未来评分映射）

评分维度包括证据等级、来源独立性、时间匹配度、推理完整度、反证覆盖度和缺口透明度。证据链评分应作为所有研究报告质量评分的基础项。
