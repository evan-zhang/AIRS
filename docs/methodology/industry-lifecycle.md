# Industry Lifecycle Analysis（产业生命周期分析）

**归属 Milestone**：M2 Methodology Core  
**适用层级**：Methodology Layer → Research Skill Layer → Score Layer  
**免责声明**：本文档仅用于产业研究方法定义，不构成投资建议，不提供个性化资产配置或价格预测。

## 1. Purpose（目的）

产业生命周期分析用于判断产业处于导入期、成长期、成熟期、衰退期或再加速阶段，并解释不同阶段下增长、竞争、利润率、资本开支和估值叙事的变化。

## 2. Theory（理论基础）

理论基础来自产品生命周期、S 曲线扩散、规模经济和竞争战略。产业阶段会影响企业行为：早期重技术验证，中期重产能和渗透率，成熟期重效率和集中度，衰退期重现金流和退出机制。

## 3. Background（背景）

AIRS 面向多行业投研，很多错误来自把早期产业按成熟行业估值，或把成熟产业按高成长叙事外推。生命周期方法为后续评分和报告提供阶段坐标。

## 4. Applicable Scenarios（适用场景）

- 用户询问行业空间、渗透率、竞争格局或成长阶段。
- 需要比较不同产业或同一产业不同环节的阶段差异。
- 需要判断政策、技术或需求是否引发再加速。

## 5. Non-applicable Scenarios（不适用场景）

- 研究对象是一次性事件，缺少产业演进维度。
- 数据不足以判断渗透率、增长率或竞争格局。
- 用户要求基于生命周期直接给出投资评级。

## 6. Inputs（输入要求）

- 产业定义和边界。
- 市场规模、渗透率、增长率、价格趋势和利润率数据。
- 竞争格局、集中度、进入退出情况。
- 技术路线、政策环境和替代品信息。

## 7. Outputs（输出规范）

- 生命周期阶段判断及证据。
- 阶段驱动因素：需求、技术、供给、政策和竞争。
- 阶段风险：过早、过热、成熟放缓或衰退误判。
- 对后续 Skill、评分和 Benchmark 的映射。

## 8. Workflow（工作流程）

1. 定义产业边界和统计口径。
2. 收集市场规模、渗透率、增长、价格和利润率数据。
3. 识别竞争格局和资本开支阶段。
4. 判断产业所处阶段，并说明阶段证据。
5. 构造反证，检验是否存在区域、环节或周期错配。
6. 输出阶段判断、置信度和跟踪指标。

## 9. Required Evidence（必需证据）

- 市场规模和增长率证据。
- 渗透率或替代率证据。
- 竞争格局、集中度或进入退出证据。
- 价格、利润率或资本开支证据。

## 10. Counter Evidence（反证要求）

- 是否存在统计口径变化导致的虚假增长。
- 是否只有局部环节成长，而非全产业成长。
- 是否存在政策补贴退坡、价格战或技术替代。
- 是否存在周期性反弹被误判为生命周期再加速。

## 11. Failure Cases（失效场景）

- 只用收入增速判断生命周期阶段。
- 忽略区域和技术路线差异。
- 把市场空间上限当作确定性增长。
- 未识别价格下降对利润的侵蚀。

## 12. Confidence（置信度评估方法）

置信度由数据口径稳定性、时间序列长度、跨指标一致性、反证强度和产业边界清晰度决定。若关键指标相互矛盾，应输出多情景判断而非单一阶段标签。

## 13. Benchmark Mapping（基准测试映射）

Benchmark 应覆盖早期高不确定性、成长高景气、成熟价格战和衰退再加速案例。标准答案检查 Agent 是否能用多指标判断阶段，并说明阶段误判风险。

## 14. Future Prompt Mapping（未来 Prompt 映射）

Prompt 应要求输入产业边界，输出阶段证据矩阵、反证矩阵、阶段结论和后续跟踪指标。禁止直接输出投资建议。

## 15. Future Skill Mapping（未来 Skill 映射）

映射到 `industry`、`hot-topic`、`valuation`、`risk` 和 `report` Skill。Industry Skill 负责阶段判断，Valuation Skill 仅使用阶段信息作为估值假设之一。

## 16. Future Score Mapping（未来评分映射）

评分维度包括渗透率空间、增长持续性、竞争强度、利润率趋势、资本开支纪律、政策依赖度和阶段置信度。评分用于研究解释，不构成买卖评级。
