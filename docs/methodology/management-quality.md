# Management Quality Analysis（管理层分析）

**归属 Milestone**：M2 Methodology Core  
**适用层级**：Methodology Layer → Research Skill Layer → Report Layer  
**免责声明**：本文档仅用于公开信息研究，不构成对个人或公司的法律评价，不构成投资建议，也不提供交易指令。

## 1. Purpose（目的）

管理层分析用于评估公司治理、战略执行、资本配置、信息披露和激励约束对企业长期经营质量的影响。该方法论要求以公开证据为基础，避免人格化猜测。

## 2. Theory（理论基础）

理论基础来自公司治理、委托代理理论和资本配置框架。优秀管理层通常体现为战略一致性、执行兑现、稳健资本开支、透明披露和股东利益协调；但管理质量必须通过行为和结果验证。

## 3. Background（背景）

在 AIRS 的公司研究中，管理层因素常影响财务质量、风险暴露和长期竞争力。M2 先定义管理层分析框架，供后续报告和评分系统使用。

## 4. Applicable Scenarios（适用场景）

- 用户要求分析公司治理、管理层能力或战略执行。
- 财报、并购、回购、融资、股权激励等事件需要管理层解释。
- 研究报告需要评估非财务质量因素。

## 5. Non-applicable Scenarios（不适用场景）

- 缺少公开信息，无法验证管理层行为。
- 用户要求评价私人品格、未经证实传闻或非公开事项。
- 用户要求把管理层评价直接转为买卖建议。

## 6. Inputs（输入要求）

- 管理层履历、任期、职责和变动记录。
- 公司战略目标、历史承诺和兑现情况。
- 资本配置行为：研发、并购、分红、回购、融资和扩产。
- 治理结构、激励机制和关联交易信息。

## 7. Outputs（输出规范）

- 管理层质量画像：战略、执行、资本配置、治理和披露。
- 承诺兑现表：历史承诺、实际结果、偏差原因。
- 风险和反证：过度扩张、关联交易、激励错配或披露不足。
- 置信度、证据来源和免责声明。

## 8. Workflow（工作流程）

1. 收集管理层和治理结构公开信息。
2. 梳理历史战略承诺和关键经营目标。
3. 对照财务结果、经营指标和资本配置行为。
4. 检查激励机制与股东、员工和债权人利益是否一致。
5. 构造反证，寻找战略失误、执行偏差或披露风险。
6. 输出管理层质量评分和不确定性。

## 9. Required Evidence（必需证据）

- 年报、公告、业绩会纪要或公开采访。
- 历史经营目标与实际结果。
- 资本开支、并购、融资、分红、回购等行为证据。
- 治理结构、独立董事、股权激励和关联交易披露。

## 10. Counter Evidence（反证要求）

- 管理层承诺是否存在未兑现或频繁变化。
- 资本配置是否损害财务稳健性。
- 激励是否鼓励短期利润或高风险扩张。
- 管理层变动是否削弱战略连续性。

## 11. Failure Cases（失效场景）

- 用个人履历替代管理质量分析。
- 把公开口号当作战略执行证据。
- 忽略行业周期和外部冲击对兑现情况的影响。
- 引用未经证实的个人传闻。

## 12. Confidence（置信度评估方法）

置信度由信息透明度、历史样本长度、承诺兑现可验证性、治理证据质量和反证强度决定。新任管理层或信息披露不足时，应降低置信度并输出观察清单。

## 13. Benchmark Mapping（基准测试映射）

Benchmark 应包含优秀执行、战略摇摆、资本配置失误和信息披露不足案例。评测 Agent 是否能用公开证据评价行为，而不是输出主观印象。

## 14. Future Prompt Mapping（未来 Prompt 映射）

Prompt 应要求提取管理层行为证据、承诺兑现表、资本配置记录和治理风险。输出不得包含人格攻击或未经证实指控。

## 15. Future Skill Mapping（未来 Skill 映射）

映射到 `financial`、`risk`、`evidence`、`committee` 和 `report` Skill。管理层分析结果可作为报告中的定性质量模块。

## 16. Future Score Mapping（未来评分映射）

评分维度包括战略清晰度、执行兑现度、资本配置纪律、治理透明度、激励一致性、风险控制和证据质量。评分不代表投资评级。
