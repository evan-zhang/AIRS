# Supply Chain Chokepoint Analysis（供应链卡脖子分析）

**归属 Milestone**：M2 Methodology Core  
**适用层级**：Methodology Layer → Research Skill Layer → Evidence Layer → Score Layer  
**免责声明**：本文档仅定义投资研究方法论，不构成投资建议，不提供买入、卖出、持有指令，也不预测证券价格。

## 1. Purpose（目的）

供应链卡脖子分析用于识别产业链中制约产能、交付、成本或技术迭代的关键环节。该方法论帮助 Research Agent 判断某一环节是否具备稀缺性、不可替代性和议价能力，并要求所有结论必须由证据链支撑。

## 2. Theory（理论基础）

理论基础来自产业组织、供应链管理和资源基础观。若某环节具备高技术壁垒、认证周期长、替代供应不足或上游资源稀缺，则该环节可能在需求上行时形成瓶颈，但瓶颈强度会随扩产、替代技术和政策干预变化。

## 3. Background（背景）

AIRS 中的供应链研究常面向 AI 服务器、半导体、创新药、新能源和机器人等复杂产业。此类产业存在多层级 BOM、长验证周期和全球分工，单看收入增速容易忽略关键材料、设备、工艺或产能环节的真实约束。

## 4. Applicable Scenarios（适用场景）

- 用户问题涉及“卡点”“瓶颈”“短缺”“国产替代”“关键供应商”“供需缺口”等关键词。
- 研究对象存在明确产业链上下游，且可拆分为材料、设备、零部件、制造、封测、渠道等环节。
- 需要解释产业景气传导、利润再分配或某一环节议价能力变化。

## 5. Non-applicable Scenarios（不适用场景）

- 研究对象没有清晰供应链结构，例如纯宏观主题或单一消费偏好变化。
- 主要矛盾来自需求不足、渠道库存或品牌竞争，而非供给约束。
- 用户要求直接荐股、价格预测或交易指令时，应拒绝该用途并转为研究框架说明。

## 6. Inputs（输入要求）

- 研究问题：必须包含产业、环节或产品范围。
- 时间范围：至少说明当前、近一年或中长期视角。
- 证据偏好：优先使用公司公告、财报、产能规划、行业协会数据、政策文件和权威研究报告。
- 约束条件：地理范围、供应商范围、技术路线和下游应用场景。

## 7. Outputs（输出规范）

- 供应链分层图谱：列出上游、中游、下游与关键节点。
- 卡点候选清单：每个候选节点说明约束类型、证据、反证和置信度。
- 瓶颈强度评分：从稀缺性、替代性、扩产难度、认证周期、议价能力等维度评分。
- 研究结论必须包含反方观点、不确定性和免责声明。

## 8. Workflow（工作流程）

1. 明确产业边界和研究对象，拆分供应链层级。
2. 收集每个环节的供需、产能、认证、成本和集中度证据。
3. 筛选潜在卡点，并排除仅由短期情绪造成的伪卡点。
4. 对候选卡点建立“约束原因 → 影响路径 → 受益/受损环节”的证据链。
5. 生成反证清单，检验替代路线、扩产计划和政策变量。
6. 输出评分、置信度和需要持续跟踪的指标。

## 9. Required Evidence（必需证据）

- 产能、出货量、良率、交期或库存数据。
- 供应商数量、市场份额、客户认证周期和技术门槛证据。
- 下游需求变化或订单能见度证据。
- 至少一个官方或原始来源证据，以及至少一个交叉验证来源。

## 10. Counter Evidence（反证要求）

- 替代供应商、替代材料、替代工艺或替代技术路线是否可行。
- 既有厂商和新进入者扩产是否会缓解瓶颈。
- 下游需求是否存在高估、推迟或取消风险。
- 政策、贸易限制或客户自研是否改变原有约束。

## 11. Failure Cases（失效场景）

- 仅凭股价表现或市场传闻推断卡点。
- 将短期交付扰动误判为长期结构性瓶颈。
- 忽略客户认证、产品规格差异和区域供应链差异。
- 未识别需求下修导致的供需反转。

## 12. Confidence（置信度评估方法）

置信度由证据等级、证据一致性、时效性、反证强度和推理链完整度共同决定。A级官方数据且多源一致时可给高置信度；若主要依赖媒体报道、专家观点或单一公司口径，应降级并明确未验证假设。

## 13. Benchmark Mapping（基准测试映射）

Benchmark 应构造包含真实产业链节点、混合强弱证据和诱导性噪声的测试用例。标准答案应检查 Agent 是否能识别关键卡点、排除伪卡点、给出反证，并避免把卡点结论转化为荐股。

## 14. Future Prompt Mapping（未来 Prompt 映射）

未来 Prompt 应拆为供应链拆解、卡点识别、证据采集、反证生成和结论合成五类。Prompt 必须强制输出证据来源、假设条件和“不可作为投资建议”的免责声明。

## 15. Future Skill Mapping（未来 Skill 映射）

对应 Skill 可包括 `supply-chain`、`chokepoint`、`evidence`、`risk` 和 `report`。Master Skill 负责选择本方法论，子 Skill 负责执行拆链、评分、反证和报告生成。

## 16. Future Score Mapping（未来评分映射）

评分维度建议包括供给紧张度、不可替代性、扩产难度、客户认证壁垒、议价能力、证据质量和反证压力。综合评分必须附带权重解释，不得被解释为投资评级。
