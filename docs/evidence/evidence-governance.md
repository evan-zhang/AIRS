# Evidence Governance（证据治理）

**归属 Milestone**：M3 Evidence Engine  
**版本**：v0.3.0  
**最后更新**：2026-07-10

**免责声明**：证据治理规范用于 AIRS 研究质量控制，不构成投资建议，不提供交易建议或收益承诺。

---

## 1. 治理目标

Evidence Governance 定义证据的责任边界、合规要求、审计要求和质量门槛，防止 AIRS 输出无来源、单边、过度确定或带有荐股倾向的研究内容。

## 2. 角色与责任

| 角色 | 责任 |
|------|------|
| Research Agent | 采集证据、生成证据卡、披露缺口 |
| Review Agent | 审查质量、调整等级、识别反证 |
| Verification Agent | 运行脚本、检查 Schema、记录 PASS/FAIL |
| Code Agent | 维护规范、模板、脚本和交付报告 |
| Human Reviewer | 对高风险报告做最终判断 |

## 3. 合规底线

所有 Evidence 相关输出必须遵守：

1. 不输出个性化投资建议。
2. 不输出买入、卖出、持有指令。
3. 不输出目标价或确定性涨跌判断。
4. 不把研究评分解释为投资评级。
5. 所有报告必须包含免责声明。
6. 对数据不确定性和来源限制进行披露。

## 4. 数据治理

### 4.1 来源治理

- 优先使用官方、原始、可复核来源。
- 对付费数据库或内部资料，应记录可复核路径或访问限制。
- 对无法公开复核的来源，应降低 evidence_level 或 confidence。

### 4.2 口径治理

- 财务数据必须记录币种、会计期间和是否调整。
- 行业数据必须记录统计范围和口径。
- 政策证据必须区分征求意见、正式发布和执行细则。

### 4.3 时间治理

- 每张证据卡必须区分 publication_time 与 collection_time。
- 对旧证据必须说明为何仍适用。
- 对实时性较强的主题应设置更新触发条件。

## 5. 审计要求

Traceability 至少记录：

- 采集者。
- 审查者。
- 来源路径。
- 处理步骤。
- 版本变更。
- 下游引用位置。

若报告结论被质疑，Verification Agent 应能从报告段落回溯到 Evidence Chain，再回溯到 Evidence Card 和原始来源。

## 6. 质量指标

建议后续 M4-M7 引入以下质量指标：

- 核心结论 A/B 级证据覆盖率。
- 反方证据覆盖率。
- Missing Evidence 披露率。
- 单源依赖率。
- 过期证据占比。
- Evidence Chain 断点数量。

## 7. 例外处理

当研究主题缺少公开证据时，Agent 可以输出低置信度观察清单，但必须：

- 明确说明证据不足。
- 不输出强结论。
- 不进入高分 Score。
- 不转化为投资建议。
- 给出后续需要验证的证据清单。
