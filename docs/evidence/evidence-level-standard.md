# Evidence Level Standard（证据等级标准）

**归属 Milestone**：M3 Evidence Engine  
**版本**：v0.3.0  
**最后更新**：2026-07-10

**免责声明**：证据等级用于研究质量控制，不构成投资建议，不构成投资评级，也不构成收益判断。

---

## 1. 等级定义

| 等级 | 名称 | 说明 | 常见来源 |
|------|------|------|----------|
| A | 原始官方证据 | 官方、原始、可复核，直接对应研究命题 | 财报、公告、监管披露、政府原文、官方统计 |
| B | 权威分析证据 | 权威机构整理或分析，有明确来源和方法 | 行业协会报告、审计后数据库、券商深度研报 |
| C | 二手报道证据 | 媒体、转述或二手摘要，需要交叉验证 | 新闻报道、会议纪要摘要、公开采访 |
| D | 观点线索证据 | 主观判断或专家观点，只能作为线索 | 专家评论、论坛观点、未披露模型观点 |
| E | 不合格证据 | 来源不明、不可验证、明显过期或与命题无关 | 无链接截图、传闻、匿名爆料 |

## 2. 等级使用规则

### A 级

A 级证据可以支撑核心结论，但仍需检查时间、口径和适用范围。多个 A 级证据如果来自同一原始文件，不得计为独立多源验证。

### B 级

B 级证据适合用于行业判断、趋势解释和交叉验证。若 B 级报告引用了 A 级来源，应在 Traceability 中记录原始来源。

### C 级

C 级证据只能作为补充或线索。若核心结论依赖 C 级证据，必须降低 confidence，并列入 missing_evidence。

### D 级

D 级证据主要用于生成反方假设、观察清单或情景分析。不得单独用于支撑高置信度结论。

### E 级

E 级证据不得进入正式 Evidence Chain，只能在审查报告中作为剔除样本说明。

## 3. 降级规则

即使来源看似权威，存在以下情况也必须降级：

- 时间过旧且未说明仍然适用。
- 数据口径不明。
- 来源存在利益冲突且未披露。
- 只引用结论，不提供方法或原始出处。
- 与研究命题相关性弱。
- 无法复核链接或文档路径。

## 4. 升级限制

证据不得仅因“被多处转载”升级。升级必须满足：

1. 独立来源。
2. 独立采集或独立方法。
3. 时间范围匹配。
4. 对同一命题提供互补信息。

## 5. 与 Confidence 的关系

Evidence Level 是来源与证据质量等级，Confidence 是该证据在当前命题下的可信程度。A 级证据也可能因为口径不匹配而只有中等 confidence；C 级证据即使内容合理，也通常不应给高 confidence。

## 6. 最低结论门槛

- 核心结论：至少 1 张 A/B 级支持证据，并有反方证据审查。
- 中间判断：至少 1 张可追溯证据。
- 反方观点：强反方观点至少需要 B 级以上证据；中等反方观点至少需要 C 级以上证据。
- 风险提示：可使用 C/D 级线索，但必须标注推测性。
