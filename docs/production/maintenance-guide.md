# AIRS Maintenance Guide（维护指南）

**版本**：v1.0.0  
**适用范围**：AIRS V1.0 生产维护  
**面向角色**：维护人员、Code Agent、Verification Agent

## 1. 维护目标

维护工作的目标是保持 AIRS 规范库长期可用、可验证、可审计。维护不应随意扩展核心能力，生产补丁应优先修复准确性、合规性、一致性和可验证性问题。

## 2. 日常维护

每次维护应执行：

- 检查 `git status`，确认工作区变更来源。
- 阅读相关里程碑 Completion Report。
- 修改前确认影响范围。
- 修改后运行对应 validate 脚本。
- 生产级维护后运行 `scripts/production_check.py`。

## 3. 周期性维护

建议每个发布周期执行：

1. 检查所有 Markdown 是否包含必要免责声明。
2. 检查所有 JSON Schema 是否可被 Python JSON parser 解析。
3. 检查 Prompt、Skill、Benchmark 的交叉引用是否存在。
4. 检查 CHANGELOG 是否记录实际变更。
5. 检查 ROADMAP 是否准确反映当前状态。
6. 检查 SECURITY 和 SUPPORT 是否仍适用。

## 4. 技术债维护

当前 V1.0 的主要技术债：

- 尚未实现真实运行时 Prompt 渲染器。
- 尚未实现 Skill 调度器。
- 尚未实现 Scorecard JSON 自动生成器。
- Benchmark 仍是分类种子和示例，不是 300+ 可执行用例全集。
- Markdown 到 JSON Schema 的自动转换链路尚未完成。

维护时不得把上述技术债描述为已经完成的能力。

## 5. 合规维护

维护人员必须持续检查：

- 是否出现“建议 + 买入”“建议 + 卖出”“目标价”“保证 + 收益”等禁止表达。
- 是否有研究报告缺少反方观点。
- 是否有评分被解释为投资评级。
- 是否有示例输出缺少免责声明。
- 是否有自动交易或交易执行暗示。

## 6. 事故处理

生产事故分级：

- P0：出现荐股、交易指令、收益承诺或严重免责声明缺失。
- P1：验证脚本误报 PASS 或关键文件缺失。
- P2：文档引用错误、模板字段遗漏、CHANGELOG 记录不完整。
- P3：格式、错字、说明不清。

P0/P1 必须立即修复并发布 patch；P2/P3 可纳入常规维护。

## 7. 免责声明

维护 AIRS 只能提高研究流程质量和系统治理质量，不能保证任何投资研究结论正确。AIRS 不构成投资建议，不提供交易指令或收益承诺。
