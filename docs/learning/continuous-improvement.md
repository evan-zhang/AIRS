# Continuous Improvement

## 1. 闭环

Continuous Improvement 把 AIRS 的一次研究执行转化为长期质量资产。闭环从 Planner、Runtime、Research Engine、Committee、Report 和 Review 产生的记录开始，经 Learning Engine 生成 Pattern、Rule 和 Optimization，再由 Review Agent 与 Verification Agent 验证，最后以受控方式进入后续版本。

## 2. 节奏

持续改进可以按 Feature、Sprint 或 Release 周期执行。高严重度问题可进入即时 Review；中低严重度问题进入批量学习窗口。所有学习结果必须进入 Completion Report 或 Self Review。

## 3. 质量门禁

改进候选必须满足来源可追溯、证据可复核、边界明确、回滚方案存在、回归脚本通过和免责声明完整六项要求。

## 4. 输出

最终输出包括学习摘要、候选规则、优化建议、治理状态、验证结果和已知缺口。
