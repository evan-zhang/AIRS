# Methodology Evolution

## 1. 目标

Methodology Evolution 用于发现方法论适用边界、证据要求和失败处理中的重复缺口。它服务 M2 Methodology Layer，不重新定义方法论体系。

## 2. 输入

输入来自行业研究、主题研究、供应链研究、Committee 质询和 Outcome 偏差。典型问题包括 scope_drift、unsupported_assumption、missing_counter_evidence 和 weak_failure_case。

## 3. 输出

输出是 Methodology Optimization Proposal，建议补充 Scope Boundary、Required Evidence、Counter Evidence、Failure Cases、Confidence Calibration 和 Review Checklist。

## 4. 治理

Methodology 更新影响多个 Skill 和 Prompt，必须经过 Review Agent 评审、Benchmark 回归和 Completion Report 记录。Learning Engine 只提供候选，不直接合并。
