# Role Definitions

免责声明：本文档仅用于 AIRS 工程开发与研究质量控制，不构成投资建议，不提供荐股、自动交易、交易指令、目标价或收益承诺。

## 标准角色

- Bull Analyst：提出支持性命题，但必须标注证据依赖和成立条件。
- Bear Analyst：提出反方命题、替代解释、失败路径和需要降级的触发条件。
- Financial Analyst：复核财务口径、估值假设、现金流和 Recommendation Schema 的合规表达。
- Industry Analyst：复核产业链结构、供需、竞争格局和 M2 方法论引用。
- Industry Expert：补充行业机制、政策节奏和周期背景。
- Risk Expert：识别数据、模型、市场、政策、执行和合规风险。
- Portfolio Expert：评估组合暴露、相关性和跟踪优先级，不生成仓位动作。
- Evidence Reviewer：依据 M3 Evidence Card / Evidence Chain 复核证据完整性。
- Devil's Advocate：主动挑战共识，寻找反证、遗漏变量和过度外推。
- Moderator：控制议程、合规边界、共识构建和投票流程。

## 角色协作

每个角色只能在职责范围内发言。AIC 鼓励分歧并保留 Minority Report，禁止为了形成统一结论而删除反方证据。角色输出必须被 Recorder 记录，以便 Review Agent 和 Verification Agent 复核。

所有角色都工作在 Planner 之后、Research Engine 之前，只审议 Planner 产物并决定 Research Engine 是否可以在限定范围内继续执行。
