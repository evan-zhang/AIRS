# Minority Report

免责声明：本文档仅用于 AIRS 工程开发与研究质量控制，不构成投资建议，不提供荐股、自动交易、交易指令、目标价或收益承诺。

## 目的

Minority Report 用于保留未被多数采纳但有实质证据或逻辑价值的少数意见。AIC 认为高质量研究必须呈现分歧，而不是用单一叙事覆盖所有风险。

## 内容

少数报告应包含提出角色、被挑战命题、反方证据、替代解释、成立条件、需要观察的触发指标和建议的复核时间。它不得包含交易动作、价格判断或收益承诺。

## 使用

Research Engine 在后续执行时必须读取 Minority Report，并在报告中保留与主结论相冲突的有效观点。Review Agent 应检查 Minority Report 是否被认真回应，Verification Agent 应检查其是否出现在 Decision Record 中。

Minority Report 必须追溯到 Planner 的原始研究范围，避免 Research Engine 在执行阶段丢失关键分歧。
