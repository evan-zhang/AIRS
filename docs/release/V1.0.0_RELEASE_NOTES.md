# AIRS v1.0.0

发布日期：2026-07-11

免责声明：AIRS 仅用于投资研究流程编排、证据追溯和质量控制，不构成投资建议，不提供荐股、交易指令、目标价或收益承诺。

## Summary

AIRS v1.0.0 is the first Stable release of the AI Investment Research System. It provides a local-first research workflow platform with CLI, REST API, Web UI, Docker deployment, APP-001 equity research workflow, Stable release gates, and production verification.

## Highlights

- CLI for init, run, demo, and validation.
- REST API with API key protection, body limit, rate limit, and redacted errors.
- Static Web UI for local research workflow control.
- Docker Compose deployment for API and Web services.
- APP-001 Equity Research App integrated with Planner, Orchestrator, Runtime, Committee, Investment Engine, Evidence/KG/Score/Report, Memory, and Learning.
- Stable release gate that prevents Mock, SKIP, Fallback, or Unknown data from counting as real production PASS.
- CI Docker Release Gate on GitHub Hosted Ubuntu runner.

## Install

```bash
git clone https://github.com/evan-zhang/AIRS.git
cd AIRS
git checkout v1.0.0
python3 cli/airs.py init
python3 cli/airs.py validate --all
```

## Docker Quick Start

```bash
cd docker
AIRS_API_KEY=change-me docker compose build --no-cache
AIRS_API_KEY=change-me docker compose up -d
curl -H "X-AIRS-API-Key: change-me" http://127.0.0.1:8765/health
```

## Known Limitations

- AIRS is not an investment advisor and does not provide trading instructions.
- Real connector availability depends on public network access and, for some providers, credentials.
- Local demo paths may use degraded data; Stable release gates expose and block degraded real-data claims.
- Runtime is a governed local workflow runtime, not a distributed multi-tenant production service.

## Security Notes

- Use `AIRS_API_KEY` before exposing API beyond localhost.
- Keep connector credentials in environment variables or local `.env` files.
- Keep `AIRS_EXPOSE_ERRORS=false` in production-like environments.
