# AIRS 系统架构

## 总体架构

AIRS 采用分层架构设计，从用户意图到研究结论，共 8 层：

```
┌─────────────────────────────────────────────────────────────────┐
│                     User Intent Layer                            │
│                    用户意图层 (研究问题)                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   Methodology Layer                              │
│              方法论层 (选择合适的研究方法)                         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Research Skill Layer                          │
│              研究技能层 (执行具体研究任务)                         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Evidence Layer                                 │
│               证据层 (采集和验证证据)                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     Score Layer                                   │
│               评分层 (量化评估)                                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   Evaluation Layer                                │
│            评估层 (反方观点与不确定性分析)                         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Report Layer                                   │
│              报告层 (生成结构化研究报告)                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   Benchmark Layer                                 │
│           基准层 (测试与质量保证)                                  │
└─────────────────────────────────────────────────────────────────┘
```

---

## 第1层：User Intent Layer

**职责**：接收和理解用户研究意图。

**输入**：
- 用户研究问题（如："分析 AI 服务器的供应链卡点"）
- 研究范围约束
- 时间要求

**输出**：
- 结构化研究意图
- 研究类型分类（供应链分析 / 产业分析 / 个股分析 / 事件分析）

**关键组件**：
- 意图识别 Prompt
- 意图 Schema
- 研究类型分类器

**目录**：`docs/overview/`

---

## 第2层：Methodology Layer

**职责**：根据研究意图选择合适的研究方法论。

**输入**：
- 结构化研究意图
- 可用方法论列表

**输出**：
- 选定方法论（如："供应链卡脖子分析"）
- 方法论执行计划

**关键组件**：
- 方法论选择 Prompt
- 方法论 Schema
- 方法论集成文档

**支持的方法论**：
- 供应链卡脖子分析
- 主题扩散分析
- 产业生命周期分析
- 政策与监管驱动分析
- 财报异常语言分析
- 新闻事件影响分析
- 反共识验证

**目录**：`docs/methodology/`, `prompts/` 各方法论子目录

---

## 第3层：Research Skill Layer

**职责**：执行具体研究任务，调用专业技能。

**输入**：
- 选定方法论
- 研究对象（如："AI 服务器产业链"）

**输出**：
- 研究任务执行结果
- 中间研究发现

**关键组件**：
- Master Skill（总控）
- 子 Skill（hot-topic, industry, supply-chain, chokepoint, financial, news, evidence, committee, valuation, risk）
- Skill Schema
- Skill 集成文档

**目录**：`skills/`

---

## 第4层：Evidence Layer

**职责**：采集、验证、关联证据，建立证据链。

**输入**：
- 研究任务执行结果
- 数据源接口

**输出**：
- 证据卡集合
- 证据链图谱
- 证据质量评分

**关键组件**：
- 证据采集 Prompt
- 证据验证 Prompt
- 证据卡 Schema
- 证据关联规则
- 证据质量评分模型

**证据类型**：
- 公开财报数据
- 行业研究报告
- 新闻报道
- 公司公告
- 政策文件
- 专家观点

**目录**：`schemas/evidence/`, `skills/evidence/`

---

## 第5层：Score Layer

**职责**：对研究对象进行量化评分。

**输入**：
- 证据卡集合
- 研究结论

**输出**：
- 多维度评分
- 评分解释
- 评分置信度

**关键组件**：
- 评分模型设计
- 评分权重体系
- 评分 Prompt
- 评分 Schema
- 评分计算规则

**评分维度**：
- 供给紧张度
- 不可替代性
- 扩产难度
- 议价能力
- 市场定价程度

**目录**：`scoring/`, `schemas/score/`

---

## 第6层：Evaluation Layer

**职责**：反方观点分析与不确定性标注。

**输入**：
- 研究结论
- 证据链
- 评分结果

**输出**：
- 反方观点清单
- 不确定性标注
- 风险提示

**关键组件**：
- 反方观点生成 Prompt
- 不确定性分析 Prompt
- 评估 Schema
- Loop/Go 评估机制

**评估维度**：
- 结论有效性
- 证据充分性
- 反方观点强度
- 不确定性等级

**目录**：`evaluation/`

---

## 第7层：Report Layer

**职责**：生成结构化研究报告。

**输入**：
- 研究结论
- 证据链
- 评分结果
- 反方观点
- 不确定性标注

**输出**：
- 结构化研究报告
- 报告元数据
- 报告质量评分

**关键组件**：
- 报告模板
- 报告生成 Prompt
- 报告 Schema
- 报告质量检查

**报告结构**：
- 核心观点
- 证据链
- 评分结果
- 反方观点
- 不确定性标注
- 风险提示
- 免责声明

**目录**：`templates/report-template.md`, `schemas/report/`

---

## 第8层：Benchmark Layer

**职责**：测试与质量保证。

**输入**：
- 研究报告
- 测试用例

**输出**：
- 测试结果
- 质量评估
- 改进建议

**关键组件**：
- 测试用例库
- 测试 Rubric
- 测试执行脚本
- 测试报告 Schema

**测试覆盖**：
- 方法论覆盖
- 行业覆盖
- Agent 能力覆盖

**目录**：`benchmark/`

---

## 横切关注点

### 数据源层

贯穿所有层的数据支撑：

- 财报数据源
- 行业数据源
- 新闻数据源
- 政策数据源
- 市场数据源

**目录**：`docs/data-sources/`

### 知识图谱层

支撑证据关联与推理：

- 产业知识图谱
- 供应链图谱
- 公司关系图谱

**目录**：`docs/knowledge-graph/`

### 治理与规范层

贯穿全流程的规范与约束：

- 开发规范
- 质量标准
- 免责声明
- 合规要求

**目录**：`docs/governance/`

## 扩展方式

### 新增方法论

1. 在 `docs/methodology/` 添加方法论文档
2. 在 `prompts/` 添加对应 Prompt
3. 在 `schemas/research/` 添加 Schema
4. 在 `benchmark/` 添加测试用例

### 新增 Skill

1. 在 `skills/` 创建 Skill 目录
2. 编写 Skill README
3. 实现 Skill 逻辑
4. 添加 Skill 测试

### 新增数据源

1. 在 `docs/data-sources/` 添加数据源文档
2. 实现数据源接口
3. 添加数据源 Schema

---

## 架构原则

1. **分层解耦**：每层独立，接口清晰
2. **证据驱动**：所有结论基于证据
3. **可测试性**：每层可独立测试
4. **可扩展性**：易于新增方法论和 Skill
5. **可解释性**：每层输出可解释
