# AIRS Examples 目录

本目录包含 AIRS 项目的使用示例。

---

## 目录结构

```
examples/
├── README.md          # 本文件
├── reports/           # 报告示例
├── evidence/          # 证据示例
├── scores/            # 评分示例
└── workflows/         # 工作流示例
```

---

## 示例分类

### 报告示例

完整的研究报告示例，展示：
- 报告结构
- 证据链
- 评分结果
- 反方观点
- 不确定性标注

**归属 Milestone**：M5

---

### 证据示例

各种类型的证据示例：
- A 级证据（财报数据）
- B 级证据（行业报告）
- C 级证据（新闻报道）
- D 级证据（专家观点）

**归属 Milestone**：M3

---

### 评分示例

各种场景的评分示例：
- 供应链卡点评分
- 产业生命周期评分
- 风险评分

**归属 Milestone**：M4

---

### 工作流示例

完整的研究工作流示例：
- 从研究意图到报告
- 证据采集流程
- 评分计算流程

**归属 Milestone**：M2

---

## 示例格式

每个示例文件应包含：

```markdown
# 示例标题

## 场景说明
[说明这个示例的场景]

## 输入
\`\`\`
[输入数据]
\`\`\`

## 输出
\`\`\`
[输出结果]
\`\`\`

## 分析
[分析输入输出的关系]

## 关键要点
[这个示例的关键要点]

---

**归属 Milestone**：M[X]
**最后更新**：YYYY-MM-DD
```

## 示例开发流程

### 1. 创建示例文件

```bash
touch examples/{category}/{example-name}.md
```

### 2. 编写场景说明

说明示例的场景和目的。

### 3. 准备输入输出

提供真实的输入和输出。

### 4. 编写分析

分析输入输出的关系。

### 5. 提炼要点

提炼关键要点。
